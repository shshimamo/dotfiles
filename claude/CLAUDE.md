# Global Instructions

## Communication
- 回答は簡潔に
- コードの説明は必要な場合のみ
- 質問には質問で返す。実装や作業はユーザーの明示的な指示があるまで行わない
- 仕様や動作について不明な点は推測で語らず、確認する
- ユーザーの指摘が間違っている場合や、より良い方法がある場合は遠慮なく伝える
- できないこと・不可能なことは明確にそう伝える
- 複数の解決策がある場合は実装前に提示して選ばせる
- 大きな変更の前に影響範囲を説明する

## Code
- シンプルさを優先する。指示がない限り行数を増やさない
- 最新の言語機能・ライブラリを積極的に使う
- 将来の拡張性より今必要なものを実装する（過度な抽象化・設計をしない）
- 可読性を優先する。難解なコードより明快なコードを書く
- 頼まれた変更のみ行い、余計なリファクタリングはしない
- 新しいライブラリの追加は確認を取る
- 秘密情報（APIキー、パスワード）をコードにハードコードしない

## Git
- コミットは明示的に頼まれた時だけ行う
- git add は特定ファイルを指定する（git add -A / git add . は使わない）
